# infra-007 CI Pipelines Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the repo's first GitHub Actions — a contract-drift gate (infra-007) and a GHCR image build/push pipeline for the `backend` and `frontend` components.

**Architecture:** Two independent workflow files under repo-root `.github/workflows/`. `ci.yml` runs the OpenAPI + frontend-types drift checks (no MySQL needed — `app.openapi()` builds the schema from routes) plus the cheap no-DB static gates, mirroring `Application/verify.sh:41-66`. `docker-images.yml` builds `backend` and `frontend` images with Buildx and pushes them to GHCR on `main`/tags using the built-in `GITHUB_TOKEN`. The VM still builds on boot — image-pull cutover is a separate later task.

**Tech Stack:** GitHub Actions, `actions/checkout@v6`, `actions/setup-python@v5`, `actions/setup-node@v4`, `docker/login-action@v4`, `docker/setup-buildx-action@v4`, `docker/metadata-action@v6`, `docker/build-push-action@v7`, `actionlint`. All actions are pinned to commit SHAs in the workflow files (version tags shown here for readability) — copy the SHA-pinned form from `.github/workflows/`, not these floating tags.

**Reference:** Design spec at `Application/docs/plans/2026-06-08-infra-007-ci-design.md`.

## Task 1: Contract-drift CI (`ci.yml`)

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/ci.yml` with exactly:

```yaml
name: ci

on:
  pull_request:
  push:
    branches: [main]

jobs:
  contracts:
    runs-on: ubuntu-latest
    env:
      # app.openapi() builds the schema from routes without a live DB. Settings only
      # requires DATABASE_URL + SESSION_SECRET to import; both are dummies here and are
      # never connected to or used to sign anything.
      DATABASE_URL: mysql+pymysql://ci:ci@localhost:3306/ci
      SESSION_SECRET: ci-dummy-session-secret-unused-1234567890
    steps:
      - uses: actions/checkout@df4cb1c069e1874edd31b4311f1884172cec0e10 # v6
        with:
          # No git push after checkout; pip install / npm ci run third-party code,
          # so don't leave GITHUB_TOKEN in .git/config for them to read.
          persist-credentials: false

      - uses: actions/setup-python@a26af69be951a213d495a4c3e4e4022e16d87065 # v5
        with:
          python-version: "3.12"

      - name: Install backend deps
        working-directory: Application/backend
        run: |
          python -m pip install --upgrade pip
          pip install -e ".[dev]"

      - name: OpenAPI drift
        working-directory: Application/backend
        run: |
          python -m app.tools.dump_openapi > /tmp/openapi.generated.json
          if ! cmp -s /tmp/openapi.generated.json ../openapi.json; then
            echo "::error::openapi.json drift — run 'python -m app.tools.dump_openapi > openapi.json' and commit"
            diff ../openapi.json /tmp/openapi.generated.json || true
            exit 1
          fi

      - name: Backend lint, types, import boundaries
        working-directory: Application/backend
        run: |
          ruff check app tests
          ruff format --check app tests
          mypy app/domain
          lint-imports

      - uses: actions/setup-node@49933ea5288caeca8642d1e84afbd3f7d6820020 # v4
        with:
          node-version: "22"
          cache: npm
          cache-dependency-path: Application/frontend/package-lock.json

      - name: Install frontend deps
        working-directory: Application/frontend
        run: npm ci

      - name: Frontend types drift
        working-directory: Application/frontend
        run: |
          cp lib/api/types.ts /tmp/types.before.ts
          npm run -s gen:types
          if ! cmp -s /tmp/types.before.ts lib/api/types.ts; then
            echo "::error::types.ts drift — run 'npm run gen:types' and commit"
            git --no-pager diff -- lib/api/types.ts || true
            exit 1
          fi

      - name: Frontend type check + lint
        working-directory: Application/frontend
        run: |
          npx tsc --noEmit
          npx eslint .
```

## Task 2: GHCR image build/push CI (`docker-images.yml`)

**Files:**
- Create: `.github/workflows/docker-images.yml`

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/docker-images.yml` with exactly:

```yaml
name: docker-images

on:
  push:
    branches: [main]
    tags: ["v*"]
  pull_request:

permissions:
  contents: read
  packages: write

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        include:
          - component: backend
            context: Application/backend
            image: ghcr.io/supporter09/pizzahust-backend
            build_args: ""
          - component: frontend
            context: Application/frontend
            image: ghcr.io/supporter09/pizzahust-frontend
            # Bake the prod same-origin API base so the published image is cutover-ready.
            build_args: |
              NEXT_PUBLIC_API_BASE_URL=/api
    steps:
      - uses: actions/checkout@df4cb1c069e1874edd31b4311f1884172cec0e10 # v6
        with:
          # Only Docker ops after checkout (login uses GITHUB_TOKEN directly);
          # no git push, so don't persist the token in .git/config.
          persist-credentials: false

      # Skip login on pull_request: forked PRs only get a read-only token and we do
      # not push from PRs anyway (build-only smoke test below).
      - name: Log in to GHCR
        if: github.event_name != 'pull_request'
        uses: docker/login-action@650006c6eb7dba73a995cc03b0b2d7f5ca915bee # v4
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - uses: docker/setup-buildx-action@d7f5e7f509e45cec5c76c4d5afdd7de93d0b3df5 # v4

      - name: Docker metadata
        id: meta
        uses: docker/metadata-action@80c7e94dd9b9319bd5eb7a0e0fe9291e23a2a2e9 # v6
        with:
          images: ${{ matrix.image }}
          tags: |
            type=sha,format=short
            type=ref,event=branch
            type=semver,pattern={{version}}
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build and push
        uses: docker/build-push-action@f9f3042f7e2789586610d6e8b85c8f03e5195baf # v7
        with:
          context: ${{ matrix.context }}
          push: ${{ github.event_name != 'pull_request' }}
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          platforms: linux/amd64
          build-args: ${{ matrix.build_args }}
          cache-from: type=gha,scope=${{ matrix.component }}
          cache-to: type=gha,mode=max,scope=${{ matrix.component }}
```
