# Admin A5-A7 Operational UX Expansion

## Summary

This PR expands the admin A5-A7 workstreams from basic tables into operational and analytical workflows.

## What Changed

### A5 - Monitor Orders

- Added default current-day filtering on the monitor orders page.
- Added a drill-down order detail modal.
- Kept operational notes attached to `order_tracking` with phase-linked note source metadata.
- Added backend support for order detail payloads and dispatch/cancel actions.

### A6 - Customer Accounts

- Upgraded customer list with search, tier filter, lock filter, and sort controls.
- Added customer rank display and last order timestamp.
- Expanded customer detail page into a richer dossier with:
  - account summary
  - loyalty snapshot
  - benefits/redeem interpretation
  - recent orders
  - top orders

### A7 - Reports

- Reworked reports page into a dashboard-style analytics view.
- Added:
  - Total Revenue
  - Total Orders
  - Avg Order Value
  - Active Customers
  - Daily Revenue chart
  - Daily Orders chart
  - Top Selling Items table
- Added a dedicated overview endpoint for dashboard payloads.

## Backend Notes

- No new customer/report tables were introduced.
- The report dashboard is computed from existing `users`, `orders`, and `order_items` relations.
- `Active Customers` uses the selected date range and counts distinct customers with delivered orders.
