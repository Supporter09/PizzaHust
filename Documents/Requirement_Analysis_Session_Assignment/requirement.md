Requirement Analysis Session
This week’s session will focus on use-case analysis. After the class, each group is expected to analyze the use cases of your own project.
Specifically, you will:
Use a relational schema to represent relationships between data entities.
Use use-case diagrams to describe interactions between different roles (actors) and the system.
Pay close attention during the feedback and correction session, so you can avoid common mistakes in your own analysis (e.g., incorrectly treating the front-end as an actor, the back-end as the system, or the database as an actor).
For next week’s presentation, each group will present these contents. Since time is limited and it is not feasible for each group to present the full set of system requirements, you should:
Select 5 most important use cases of your system.
Ensure diversity of roles (actors) across these use cases.
Provide detailed use-case specifications for each selected use case.
The template for use-case specifications has been shared in the General channel.
Please prepare carefully to ensure an effective and focused presentation.
