\section{Analysis}

\subsection{General Use Case Overview}

This section presents the use case analysis for the \systemname\ Online Ordering System. The scope is limited to a web-based MVP that can be delivered within 3.5 months while still satisfying the mandatory requirements described in the project topic. The MVP supports online ordering, pizza customization, combo promotions, menu browsing, cash on delivery, loyalty points for registered customers, order preparation, basic order tracking, third-party delivery integration, and basic reporting for the store owner.

The system includes five primary actors and one external supporting actor:

\begin{description}
    \item[Guest:] A visitor who can browse menus, customize products, place a cash-on-delivery order, track an order, and request AI-based menu suggestions without creating an account.
    \item[Customer:] A registered user who is a specialization of Guest. Therefore, the customer inherits all guest capabilities and additionally can log in, manage profile information, view order history, earn loyalty points, and redeem points during checkout.
    \item[Admin / Store Owner:] The business operator who manages pizzas, toppings, side dishes, categories, combo campaigns, order supervision, customer support, and sales reports.
    \item[Kitchen Staff:] Internal staff members who receive confirmed orders, prepare them, update preparation progress, and mark orders as ready for delivery.
    \item[AI Recommendation Service:] An external or internal AI module that generates simple menu and combo suggestions based on the customer's preferences, budget, and group size.
    \item[Third-Party Delivery Service:] An external delivery platform integrated through API to receive delivery requests and return delivery status updates to the system.
\end{description}

\subsubsection{Use Case Relationships}

The main relationships among use cases are summarized below:

\begin{itemize}
    \item \textbf{Customer} is modeled as a specialized actor that inherits all use cases available to \textbf{Guest}.
    \item \textbf{Customize Pizza} includes selecting pizza size and crust type, and it optionally extends to adding extra toppings.
    \item \textbf{Place COD Order} includes providing delivery information, calculating the fixed delivery fee, and confirming cash on delivery.
    \item \textbf{Redeem Points for Discount} extends \textbf{Place COD Order} and is available only to authenticated customers with sufficient loyalty points.
    \item \textbf{Get AI Menu Recommendation} includes the service use case \textbf{Generate Personalized Menu Recommendation}.
    \item \textbf{Mark Order Ready for Dispatch} includes \textbf{Request Delivery Service} when the order is packed and ready for handoff.
    \item \textbf{Track Order} includes \textbf{Synchronize Delivery Status} when the order has already been sent to the third-party delivery service.
\end{itemize}
